<important_context>
- **P2D `fill(int)` pitfall**: Always use `fill(r,g,b)` or `fill(r,g,b,a)`. Never `fill(int)` with a packed color value.
- **P2D `pushStyle/popStyle` pitfall**: Can cause OpenGL state desync; removed from world/component renderers, but may remain in `TdFlowController.drawHudBackdrop()` and `TdGameWorld.drawPlacementGhost()` (using `app.` directly).
- **Screen layer rule**: `renderLayer >= 100` bypasses Camera2D but is still scaled by DisplayManager. Used for Minimap (110).
- **Mouse coordinate chain**: Actual mouse → `displayManager.actualToDesign()` → `camera.screenToWorld()` → world coords. Applied in all input handlers.
- **Camera2D transform order**: `translate(center) → scale(zoom) → rotate → translate(-pos)`. Viewport size must match design resolution (updated on window resize).
- **`addComponent(Component)` returns void** in p5engine API; cannot chain `ec = go.addComponent(new ...)`.
- **Scene.render()** collects Renderables into `worldCommands` (layer < 100, with camera transform) and `screenCommands` (layer >= 100, no camera). Culling is applied to world layers using `camera.getViewport()` vs `go.getRenderBounds()`.
- **DisplayManager** automatically wraps rendering in `begin/end` inside `P5Engine.render()`. All Scene rendering should happen in design-resolution coordinates.
- **TweenManager**: UI animations use `setUseUnscaledTime(true)` so menus work during pause.
- **ProcessingRenderer**: implements `IRenderer` wrapping `applet.g`. Recently recompiled to ensure `getGraphics()` returns the live `applet.g` reference.

```java
            setFillRGB(g, viewportColor);
            g.rect(vx, vy, vw, vh);
        }

        // Objects
        if (showObjects && scene != null) {
            setFillRGB(g, objectColor);
            for (GameObject go : scene.getGameObjects()) {
                if (!go.isActive()) continue;
                if (go.getName().equals("ship")) {
                    Vector2 pos = go.getTransform().getPosition();
                    float mx = ox + (pos.x - worldBounds.x) * minimapScale;
                    float my = oy + (pos.y - worldBounds.y) * minimapScale;
                    g.rect(mx - 4, my - 4, 9, 9);
                }
            }
        }
    }

    /** P2D's fill(int) is unreliable with negative ARGB values; use 3-float fill instead. */
    private static void setFillRGB(PGraphics g, int c) {
        float r = ((c >> 16) & 0xFF);
        float gr = ((c >> 8) & 0xFF);
        float b = (c & 0xFF);
        g.fill(r, gr, b);
    }

    // ── Interaction ──

    /**
     * Check whether a design-resolution point lies inside the minimap widget.
     */
    public boolean contains(float designX, float designY) {
        return designX >= x && designX <= x + w && designY >= y && designY <= y + h;
    }

    /**
     * Convert a design-resolution point inside the minimap to world coordinates.
     *
     * @param designX design-resolution X (e.g. from {@link DisplayManager#actualToDesign})
     * @param designY design-resolution Y
     * @return world position; clamped to the world bounds
     */
    public Vector2 minimapToWorld(float designX, float designY) {
        if (worldBounds == null) return new Vector2(0, 0);

        float scaleX = w / worldBounds.width;
        float scaleY = h / worldBounds.height;
        float minimapScale = Math.min(scaleX, scaleY);

        float drawW = worldBounds.width * minimapScale;
        float drawH = worldBounds.height * minimapScale;
        float ox = x + (w - drawW) * 0.5f;
        float oy = y + (h - drawH) * 0.5f;

        float nx = (designX - ox) / Math.max(1f, drawW);
        float ny = (designY - oy) / Math.max(1f, drawH);
        nx = Math.max(0f, Math.min(1f, nx));
        ny = Math.max(0f, Math.min(1f, ny));

        return new Vector2(
            worldBounds.x + nx * worldBounds.width,
            worldBounds.y + ny * worldBounds.height
        );
    }
}
```

## How `render()` Works

1. **Early exit** if `worldBounds` is not set.
2. **Compute scaling** to fit the world bounds into the minimap rectangle (`x, y, w, h`), preserving aspect ratio and centering the result (`ox, oy`).
3. **Set drawing mode**: `rectMode(CORNER)` + `noStroke()`.
4. **Draw layers in order**:
   - **Background** fill over the entire widget rect.
   - **World border** drawn as 4 filled strips (top, bottom, left, right) around the scaled world area.
   - **Viewport rectangle** (green by default) showing where the camera is looking, if `showViewportRect` is true.
   - **Objects** (cyan by default): iterates all `GameObject`s in the scene and draws a 9×9 px square for any object named `"ship"`.
5. **Color helper**: Uses a private static method `setFillRGB(PGraphics, int)` which unpacks an ARGB `int` into `R, G, B` floats and calls `g.fill(r, gr, b)`. This was specifically added to work around P2D renderer issues where `fill(int)` with ARGB values is unreliable. `pushStyle()/popStyle()` are intentionally **not used** for P2D compatibility.
