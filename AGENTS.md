# p5engine Agent 工作规则

> 本文件定义所有 AI Agent（Kimi、Cursor、Claude 等）在参与本项目时必须遵循的工作流程与规范。

## 1. 计划管理（Plan Management）

### 1.1 强制目录
**所有制定的开发计划必须放入项目根目录的 `plans/` 文件夹中。**

- 不允许将计划文件散落在项目根目录、桌面、临时文件夹或聊天记录中。
- 如果 `plans/` 目录不存在，必须先创建它。

### 1.2 文件命名规范
计划文件必须使用以下命名格式：

```
{主题简述}_{8位随机hash}.plan.md
```

- **主题简述**：用英文小写和连字符描述计划核心内容，如 `tween-system`、`ui-theme-alpha`、`level-select-menu`
- **随机 hash**：8 位小写十六进制，用于避免文件名冲突
- **扩展名**：必须为 `.plan.md`

**示例：**
```
tween-alpha-fix_a3f7b2d1.plan.md
main-menu-animation_8e2c91f4.plan.md
```

### 1.3 计划文件内容结构
每个 `.plan.md` 必须包含以下章节：

```markdown
# 计划标题

## 目标
一句话描述本次计划要达成的目标。

## 背景
当前状态、问题描述或需求来源。

## 实施步骤
1. 具体步骤一
2. 具体步骤二
3. ...

## 验收标准
- [ ] 标准一
- [ ] 标准二

## 相关文件
- `src/main/java/.../Foo.java`
- `examples/Bar/Bar.pde`
```

### 1.4 计划生命周期
- **创建**：开始新任务前，先在 `plans/` 中创建计划文件。
- **更新**：实施过程中如有步骤变更，必须同步更新计划文件。
- **完成**：任务完成后，在计划文件顶部添加 `> **状态：已完成** ✅` 标记。
- **归档**：已完成的计划文件保留在 `plans/` 中，不要删除，作为项目历史记录。

### 1.5 执行操作
- 永远不允许自行删除任何文件，需要删除操作必须询问用户。

## 2. 代码规范

### 2.1 最小修改原则
- 只修改与当前任务直接相关的代码。
- 不要顺便格式化、重构或"优化"无关文件。

### 2.2 编译验证
- 每次修改 Java 源码后，必须运行 `compile-jar.ps1` 重新编译 `p5engine.jar`。
- 每次修改示例 PDE 后，必须用 Processing CLI (`processing.exe cli --sketch=... --build`) 验证编译通过。
- **禁止将未编译验证的代码提交给用户。**

### 2.3 PDE 语法注意
- Processing 的 PDE 预处理器不支持 Java 静态字段作为方法引用。
- **正确**：`Ease::outQuad`
- **错误**：`Ease.outQuad`（会被预处理器错误解析）

## 3. 文件与目录操作

### 3.1 工作目录边界
- 默认工作目录为项目根目录 `E:\projects\kilo\p5engine`。
- 所有文件操作（创建、修改、删除）默认在此目录内进行。
- **禁止**访问或修改用户桌面、`C:\Users\...\Documents` 等个人目录，除非用户明确要求。

### 3.2 输出目录
- 引擎构建输出：`library\p5engine.jar`
- CLI 示例构建输出：`{sketch}\_build_cli\` 或 `{sketch}\build\`
- 计划文件：`plans\*.plan.md`

## 4. 沟通与汇报

### 4.1 截图验证
- UI/动画类修改完成后，运行后如果用户想要截图验证视觉效果，那么执行截图，否则不执行。
- 截图保存到示例目录或临时目录，供用户确认。

### 4.2 总结文档
- 涉及架构调整、Bug 修复或功能新增的任务完成后，应在 `docs/` 目录下创建简短的 Markdown 总结文档。
- 文档应包含：问题描述、根本原因、修复内容、验证结果。

## 5. 版本与兼容性

- **Processing 版本**：4.5.2
- **JDK 版本**：17
- **目标平台**：Windows（开发环境）
- 引入新依赖前必须征得用户同意。
